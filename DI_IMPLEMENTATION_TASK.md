# Dependency Injection Implementation Task
## Learning Exercise for Interns

### 📋 Overview
This task demonstrates how to implement Dependency Injection (DI) using the `get_it` and `injectable` packages in a Flutter Clean Architecture project. The Medicine Reminder App has been successfully refactored to use DI for better testability, maintainability, and loose coupling.

---

### 🎯 Learning Objectives

By completing this task, you will learn:

1. **What is Dependency Injection?**
   - Understanding the Inversion of Control (IoC) principle
   - Why DI is important in software architecture
   - Benefits: testability, maintainability, loose coupling

2. **Using GetIt Service Locator**
   - How to set up and configure GetIt
   - Registering dependencies (singleton, factory, lazy singleton)
   - Resolving dependencies at runtime

3. **Using Injectable Code Generation**
   - Adding `@injectable` annotations
   - Running `build_runner` to generate DI code
   - Understanding generated code structure

4. **Refactoring Existing Code**
   - Identifying tight coupling issues
   - Replacing manual instantiation with DI
   - Maintaining Clean Architecture principles

### 🛠️ Implementation Steps

#### Step 1: Add Dependencies
```bash
flutter pub add get_it injectable
flutter pub add --dev injectable_generator build_runner
```

#### Step 2: Add Injectable Annotations
Add `@injectable` to classes that should be managed by DI:

```dart
// Use case example
@injectable
class AddMedicine {
  final MedicineRepository repository;
  AddMedicine(this.repository);
  // ...
}

// Repository example (bind to interface)
@Injectable(as: MedicineRepository)
class MedicineRepositoryImpl implements MedicineRepository {
  final HiveDataSource localDataSource;
  MedicineRepositoryImpl(this.localDataSource);
  // ...
}
```

#### Step 3: Create DI Container
```dart
// lib/core/di/injection_container.dart
import 'package:get_it/get_it.dart';
import 'package:injectable/injectable.dart';
import 'injection_container.config.dart';

final getIt = GetIt.instance;

@injectableInit
void configureDependencies({String environment = 'prod'}) {
  getIt.init(environment: environment);
}
```

#### Step 4: Generate DI Code
```bash
dart run build_runner build --delete-conflicting-outputs
```

#### Step 5: Configure in main.dart
```dart
void main() async {
  // ... initialization code ...
  
  // Register manual dependencies
  getIt.registerSingleton<Box<MedicineModel>>(medicineBox);
  getIt.registerSingleton<Box<HistoryModel>>(historyBox);
  
  // Configure DI
  configureDependencies();
  
  runApp(const MedicineReminder());
}
```

#### Step 6: Use DI in Widgets
```dart
// Before (manual)
BlocProvider(
  create: (_) => MedicineCubit(
    addMedicineUseCase: AddMedicine(repository),
    // ...
  ),
)

// After (DI)
BlocProvider(
  create: (_) => MedicineCubit(
    addMedicineUseCase: getIt<AddMedicine>(),
    saveHistoryUseCase: getIt<SaveHistory>(),
    // ...
  ),
)
```

### 🔍 Key Concepts Explained

#### @injectable Annotation
Marks a class for automatic registration in the DI container.

```dart
@injectable
class MyClass {
  // Dependencies are automatically injected
}
```

#### @injectable(as: InterfaceType)
Binds an implementation to an interface/abstract class.

```dart
@Injectable(as: MedicineRepository)
class MedicineRepositoryImpl implements MedicineRepository {
  // ...
}
```

#### getIt<T>()
Resolves a dependency from the container.

```dart
final myService = getIt<MyService>();
```

#### Registration Types
- **Factory**: Creates a new instance each time
- **Singleton**: Creates once, returns same instance always
- **LazySingleton**: Creates on first use, then reuses

### 🐛 Common Issues & Solutions

#### Issue: "The function '$initGetIt' isn't defined"
**Solution**: Make sure `build_runner` has generated the config file:
```bash
dart run build_runner build --delete-conflicting-outputs
```

#### Issue: "depends on unregistered type"
**Solution**: Either:
- Add `@injectable` to the class
- Register manually with `getIt.registerSingleton<T>()`

#### Issue: Generated code doesn't update
**Solution**: Delete `.dart_tool/build` and rebuild:
```bash
rm -rf .dart_tool/build
dart run build_runner build --delete-conflicting-outputs
```

### 🎓 What to Look for in Code Review

When reviewing DI implementation:

1. **Are all dependencies injected?** (no `new` keyword for dependencies)
2. **Is the container configured before first use?**
3. **Are manual registrations necessary?** (can they be automated?)
4. **Is the generated config file checked in?** (YES)
5. **Are there circular dependencies?** (avoid!)
6. **Is the code more testable?** (should be!)
